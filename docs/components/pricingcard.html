PricingCard

Subscription pricing card component

Import

import { PricingCard } from 'mybu-expo-ui';

Description

The PricingCard component displays subscription pricing options with support for different package types. It includes a "Best Value" tag for featured packages and skeleton loading states.

Props

Prop Type Required Description
packageType string Required Type of package (e.g., "monthly", "annual", "lifetime")
priceString string Required Formatted price string to display
onPress () => void Required Callback when the card is pressed
packages Record<string, PackageInfo> Optional Package information for title, description, and period label
theBestValueText string Optional Custom text for the "Best Value" tag
skeleton number Optional Number of skeleton cards to show

PackageInfo Structure

{
  title: string,        // Package title
  description: string,  // Package description
  periodLabel: string  // Period label (e.g., "/month")
}

Usage Example

import { PricingCard } from 'mybu-expo-ui';

const packages = {
  monthly: {
    title: "Monthly",
    description: "Billed monthly",
    periodLabel: "/month"
  },
  annual: {
    title: "Annual",
    description: "Save 40%",
    periodLabel: "/year"
  },
  lifetime: {
    title: "Lifetime",
    description: "One-time purchase",
    periodLabel: ""
  }
};

const MyComponent = () => (
  <>
    <PricingCard
      packageType="monthly"
      priceString="$9.99"
      packages={packages}
      onPress={() => handlePurchase('monthly')}
    />
    <PricingCard
      packageType="annual"
      priceString="$59.99"
      packages={packages}
      theBestValueText="Best Value!"
      onPress={() => handlePurchase('annual')}
    />
    <PricingCard
      packageType="lifetime"
      priceString="$149.99"
      packages={packages}
      onPress={() => handlePurchase('lifetime')}
    />
  </>
);

// Skeleton loading
<PricingCard
  packageType="monthly"
  priceString=""
  skeleton={3}
  onPress={() => {}}
/>

Features

  • Best Value Tag: Highlight featured packages
  • Flexible Package Types: Support for any package type string
  • Custom Styling: Per-package styling based on type
  • Skeleton Loading: Multiple skeleton placeholders
  • Touchable: Entire card is pressable
